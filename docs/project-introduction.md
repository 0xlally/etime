# 我做了一个开源时间工作台：ETime

如果你也试过很多时间管理工具，可能会有一种微妙的疲惫：记录本身变成了另一件需要坚持的事。

ETime 想解决的不是“如何把每一分钟都管起来”，而是更朴素的一件事：让开始更轻，让复盘更清楚，让长期目标真的有反馈。

项目地址：<https://github.com/0xlally/etime>  
在线体验：<http://time.lally.top>

## 它是什么

ETime 是一个自托管的时间管理、目标跟踪与复盘工具。你可以把它当成一个个人时间工作台：

- 开始一段计时。
- 补录一段已经完成的时间。
- 把高频事项做成快捷卡片。
- 在日历里安排今天、这周或未来某天。
- 设置每日、每周、每月目标。
- 看统计、热力图、目标进度和复盘报告。
- 生成一张适合分享的复盘海报。
- 和小组成员一起打卡、分享今日状态。

它有 Web 端，也有基于 Capacitor 的 Android 端。后端是 FastAPI，数据放在 PostgreSQL，适合自己部署，也适合二次开发。

## 一点就开始，而不是先配置半天

ETime 的首页是计时工作台。你可以选择分类后开始实时计时，也可以直接点快捷开始卡片，比如“阅读 25 分钟”“英语 30 分钟”。

快捷模板会记住分类、固定时长、备注模板、颜色和排序。固定时长到点后，Web 和 Android 前台都会提醒你继续或结束。

建议截图：`/timer` 页面，展示快捷开始、目标进度和实时计时区域。

## 允许生活不完美：离线与补录

很多时间记录最后失败，不是因为人不自律，而是因为真实生活总会打断流程。

ETime 支持：

- 离线开始/停止计时。
- 刷新或重开后恢复本地计时状态。
- 联网后自动同步待上传记录。
- 手动补录历史时间。
- 用 `client_generated_id` 做幂等去重，避免重复同步生成重复 session。

这让它更像一个能跟上现实节奏的工具，而不是只适合完美流程的表格。

## 目标不是口号，而是每天看得见的进度

ETime 的目标引擎支持日、周、月和“明天”目标。你可以设定目标时长，也可以指定目标只统计某些分类。

目标页会展示：

- 当前目标进度。
- 剩余时长。
- 连胜和最佳连胜。
- 完成率。
- 时间债务。
- 补偿与奖惩记录。

这部分适合用来做学习计划、备考计划、健身记录、自由职业工时管理，或者任何需要长期积累的事情。

建议截图：`/targets` 页面，展示目标卡片和目标仪表盘。

## 计划日历：把任务放进时间里

ETime 不是完整项目管理软件，但它提供了刚刚好的计划日历：

- 月/周/日视图。
- 待安排池。
- 新增和编辑事项。
- 预计耗时。
- 临近提醒。
- 完成事项。
- 把计划事项转成时间记录。
- 从计划直接开始计时。

它的思路是：不要把一天塞满，而是给重要事情找一个位置。

建议截图：`/planner` 页面，展示日历、待安排池和今日摘要。

## 复盘不是翻账，而是看见节奏

ETime 提供统计页、热力图、日报和周报：

- 分类占比告诉你时间去了哪里。
- 热力图告诉你节奏是否稳定。
- 日报/周报串联目标达成、分类趋势和时痕。
- Markdown 导出方便沉淀到博客、Notion、Obsidian 或周报里。

对于长期任务来说，复盘最大的价值不是审判自己，而是看见模式：什么时候状态好，什么事情总是拖延，哪类投入正在变多。

建议截图：`/classification`、`/heatmap`、`/review` 页面。

## 分享海报：把努力做成一张安静的卡片

ETime 有一个分享卡片页面，可以生成今日、本周或本月复盘海报。

它会聚合：

- 总时长。
- 分类占比。
- 目标完成状态。
- 连续记录天数。
- 热力图预览。

隐私模式会隐藏真实分类名，也可以隐藏总时长。Web 端可以导出 PNG，Android 前台也复用同一套生成能力。

建议截图：`/share` 页面，展示三种海报样式和隐私模式。

## 小组协作：轻量，但够用

ETime 的小组功能现在是 MVP 版本，重点是轻量协作：

- 创建小组并生成邀请码。
- 通过邀请码加入。
- 查看成员列表。
- 发送文本消息。
- 分享今日状态。
- 分享复盘卡片摘要。

它适合自习小组、学习搭子、考研/考公打卡群、开发者结伴做项目，也适合朋友之间温和互相提醒。

建议截图：`/groups` 页面，展示邀请码、成员预览、消息区和分享按钮。

## Android 端：同一套体验，带到手机上

Android 端使用 Capacitor 复用前端。打包时可以指定真实后端地址：

```powershell
cd frontend
$env:VITE_API_BASE_URL = 'https://your-domain.example/api/v1'
npm run android:sync
cd android
.\gradlew.bat assembleRelease
```

目前仓库没有内置 release keystore。正式发布前需要配置签名，生成签名 APK 或 AAB。

## 技术实现

ETime 的技术栈比较直白：

- 前端：React、Vite、TypeScript、Tailwind CSS、Recharts。
- 后端：FastAPI、SQLAlchemy、Alembic。
- 数据库：PostgreSQL。
- 部署：Docker Compose、Nginx。
- 移动端：Capacitor Android。
- 认证：JWT access token + refresh token。
- 邮件：SMTP 找回密码。

安全默认值上做了几件事：

- 不内置默认管理员。
- Docker Compose 强制要求数据库密码。
- 后端日志脱敏数据库连接串。
- Docker 镜像不复制 `.env`。
- Android 禁止明文 HTTP 和应用备份。
- 自律模式密码使用 PBKDF2 派生存储。

## 适合谁

ETime 适合这些人：

- 想自托管数据，不想把长期记录放在封闭平台里。
- 想记录学习、工作、健身、写作或备考投入。
- 需要目标反馈，而不只是一个秒表。
- 想把复盘导出成 Markdown 或图片。
- 想和小组一起轻量打卡。
- 想基于 FastAPI + React 二次开发自己的时间系统。

一句话版：

> ETime 是一个开源自托管时间工作台：计时、计划、目标、统计、复盘、分享和小组协作，都放在一个安静清晰的系统里。

短介绍版：

> ETime 帮你把每天的投入记录下来，把长期目标拆成可见进度，并在复盘时生成清晰的统计、热力图和分享海报。它支持 Web 与 Android，前后端开源，可用 Docker Compose 自托管。

## 最后

我希望 ETime 是一种温和的工具：它不逼你变成机器，也不把自律做成焦虑。它只是尽量降低开始的阻力，把投入变得可见，把复盘变得轻一点。

如果你正在做一个长期目标，或者想拥有自己的时间数据，欢迎试试看。
